# 愛知工業大学 陸上競技部 公式サイト — 更新マニュアル

このサイトは **プログラミングの知識がなくても更新できる** ように作ってあります。
ふだんの更新は、`content/` フォルダの中の **メモ帳で開けるテキストファイル（.txt）** を
書き換えるだけです。HTMLやプログラムのファイルは基本的にさわりません。

> ⚠️ **とても大事な注意（先に読んでください）**
> このサイトは「テキストファイルを読み込んで表示」する仕組みです。
> そのため、HTMLファイルを **ダブルクリックして直接開くと、内容が表示されません**。
> 必ず次のどちらかで確認してください。
> - **公開後の Netlify のURL** で見る（本番。これが基本）
> - パソコンで確認したいときは **VS Code の「Live Server」** などの簡易サーバー経由で見る
>   （`127.0.0.1:5500` のような形で開く）
> ※ 今までの運用と同じです。特別な準備は増えていません。

## ✏️ ふだんの更新（content フォルダの .txt を書き換える）

### 共通のルール
- 各行の **「：」の右側だけ** を書き換えます。項目名（左側）や行の順番は変えません。
- **「#」で始まる行はメモ** です（サイトには表示されません）。消しても構いません。
- 書き換えたら **上書き保存** し、あとで Netlify に再アップロードすると反映されます。

### ① トップの文言・スローガン・SNS・連絡先 → `content/site.txt`
- ヒーロー（トップ背景の上の文字）、スローガン、X・Instagram のURL/アカウント名、連絡先メールを変更できます。
- **連絡先メールは迷惑メール対策のため「前半（@の前）」「後半（@の後ろ）」に分けて書きます。**
  例：`example.taro@gmail.com` → 前半に `example.taro`、後半に `gmail.com`。
  サイト上では自動的に組み立てて普通のメールアドレスとして表示されます
  （完成形をファイルに書かないことで、アドレス収集ボットに拾われにくくしています）。
- SNSのURLを変えると、**全ページのフッターのリンクも自動で切り替わります**。
- スローガンで金色にしたい文字は「スローガン強調文字」にカンマ区切りで書きます（例：`王, 奪`）。

### ② 監督紹介 → `content/coach.txt`
- 氏名・役職・写真・一言・経歴を変更できます。
- 経歴は「経歴：」の下に **1行1項目** で書きます（行を足す・消すだけ）。
- 写真は `images/` に置き、ファイル名だけ書きます（例：`coach.png`）。

### ③ 部員紹介 → `content/members.txt`
- 1人分を「氏名：」から書き、次の人との間に **ハイフン3つ `---` を1行** 入れて区切ります。
- 並び順は自動で「学年（4→1年）→ 記載順」に整います。ファイル内の順番は気にしなくてOK。
- 見出しの人数（例：長距離◯名）も、書いた人数に合わせて **自動で表示** されます。
- 写真は `images/members/` に置き、ファイル名だけ書きます（無ければ灰色の人型が出ます）。
- **「出身県」「学科」** は、部員カードを **クリックしたときだけ** 表示される項目です。
  まだ分からない場合は空欄のままでOK（空欄の項目は表示されません）。
- **「役職」** に何か書くと（例：`主将` / `副将` / `主務` など）、
  部員カードの **右上** と、クリックで開く拡大表示の **左上** に
  すりガラス風の金色バッジで表示されます。役職が無い人は空欄のままでOK（何も表示されません）。
- **「一言」** は、いま **短距離ブロックだけサイトに表示されない設定** になっています
  （文章は `members.txt` に残っているので、設定を戻せばそのまま出ます）。
  また、**一言が空欄の人**はカード・拡大表示のどちらにも何も出ません（`「」` は出ません）。

> 💡 部員カードは **クリックすると拡大表示** され、出身県・学科を含む詳細が出ます。

> 🔧 **短距離の「一言」を表示に戻したいとき（1か所だけ）**
> `js/content-loader.js` の先頭にある次の行を書き換えます。
> ```js
> const COMMENT_HIDDEN_BLOCKS = ['短距離'];   // ← 今の設定（短距離だけ隠す）
> const COMMENT_HIDDEN_BLOCKS = [];           // ← 全員の一言を表示する
> ```
> `['短距離', '長距離']` のように並べれば、複数ブロックを隠すこともできます。
> ブロック名は `members.txt` の「ブロック：」と同じ文字で書いてください。

### ④ 駅伝 歴代最高記録 → `content/ekiden.txt`
- 大会は2つ（【大会1】【大会2】）。それぞれ大会名・記録・順位・メンバーを変更できます。
- メンバーは「メンバー：」の下に **1人1行**、`区分 ｜ タイム ｜ 氏名 ｜ 学年` の順で書きます。
  （「｜」は全角でも半角 `|` でもOK。人数は増減自由）

### ⑤ 練習曜日 → `content/schedule.txt`
- 練習スケジュールページの **カレンダーの下のボックス** に表示される内容です。
- 「長距離：」「短距離：」の右側に曜日を自由な書き方で書きます（例：`月・火・水・金・土`）。
- 行を足せば項目も増やせます（例：`マネージャー：…`）。「タイトル」「補足」以外の行はすべて表示されます。
- 「補足」は下に小さく出る注意書きです。空欄なら表示されません。

## 📰 ニュースを追加・整理する

### 一覧にニュースを1件足す（最短ルート・メモ帳だけでOK）

`content/news.txt` を開き、次の形のブロックを1つ書き足して、
前後を「---」で区切るだけです。**書く場所は上でも下でもOK**
（ページでは自動的に日付の新しい順に並びます）。

```
日付：2026-07-01
カテゴリ：大会結果
タイトル：第○○回○○大会 結果報告
リンク：/records/results/
---
```
- カテゴリは `お知らせ / 大会結果 / 駅伝 / 活動報告` のどれか。
- リンクは、無ければ空欄。個別記事に飛ばすなら `/news/フォルダ名/`。

### 個別の記事ページも作る（Claude活用・HTML不要）

詳しい手順と記入例は **`content/news-example/はじめにお読みください.txt`** にあります。
記入用の空テンプレは **`content/news-new/`** にあります（コピーして使う）。

かんたんな流れ：
1. 写真を `images/news/` に置く（ファイル名は半角英数字）。
2. `content/news-new/` の `記事.txt`・`配置.txt` をコピーして記入する。
3. その2ファイルと `news-template.html` を Claude に渡し、写真のファイル名を伝えて
   「ニュース記事ページを完成させて」と依頼する。
4. Claude が **完成した `news/〔フォルダ名〕/index.html`** と、
   **一覧に載せるための `content/news.txt` のブロック** を作ってくれる。
5. できたファイルと写真を所定の場所に置き、Netlify に再アップロードで公開。

> 💡 **写真の見せ方（自動で選ばれます）**
> メイン写真が **縦長2枚** のときは、中央にきれいに2枚並べたグリッド表示になります。
> 写真が **3枚以上**、または **横長が2枚以上** のときは、指でスワイプして切り替えられる
> **カルーセル表示** になります。この判定は Claude が行うので、
> 配置.txt に写真を書いて渡すだけでOKです（向きが分かるように伝えると確実です）。

### 記事が「公開される」条件（重要）

次の2つがそろってはじめて一覧に表示されます：
1. 記事ページ本体を `news/フォルダ名/index.html` に置く
2. `content/news.txt` にそのブロック（リンク＝ `/news/フォルダ名/`）を追加する

作りかけの記事は **`news/_drafts/`** に入れておけばサイトには表示されません
（詳しくは `news/_drafts/このフォルダの使い方.txt`）。

### 古いニュースの整理（1年で自動非表示）

- 日付が1年以上前の記事は、ニュースページで **自動的に非表示** になります
  （データが勝手に消えることはありません）。
- 削除してよい記事の一覧は、ブラウザで **`/news/maintenance/`** を開くと
  確認できます（ナビには出ない担当者用ページ）。表示されたブロックを
  `content/news.txt` から消せば整理完了です。
- 手作業が面倒なときは、Claudeに `content/news.txt` を渡して
  **「1年以上前の古いニュースを整理してください」** と頼めば、
  整理済みのファイルと、削除してよい記事フォルダの一覧を作ってもらえます。

### 「NEW」バッジ（自動）

- 日付が **本日から7日以内** のニュースには、金色の「**NEW**」バッジが
  自動で付きます（トップページの最新情報・ニュース一覧ページの両方）。
- 特別な操作は不要です。日付だけで自動判定し、**7日を過ぎると自動で消えます**。

### ニュース一覧ページのページ送り（自動）

- ニュース一覧ページ（`/news/`）は **1ページ10件** 表示です。
- 記事が11件以上（＝直近1年以内に10件超）になると、一覧の下に
  **ページ送りボタン（‹ 1 2 3 … ›）** が自動で表示されます。
- 10件以下のときはボタンは出ません（従来と同じ見た目）。

## 📊 大会結果ページの設定（年度ごとのスプレッドシート）

大会結果は、**年度ごとに1つのGoogleスプレッドシート** から自動表示します。
スプレッドシート内の **シート（タブ）1枚 = 1大会** です。

### シート（タブ）名の付け方 ★重要
```
大会名〔半角スペース〕開催日｜開催場所
例）第2回トヨタ紡織記録挑戦会 2026年3月28日｜ウェーブスタジアム刈谷
```
- **大会名の中に半角スペースを入れない**でください（最初の半角スペースで大会名と日付を区切ります）。
- 日付・場所が無い大会は「大会名」だけでもOK（`第92回東海学生…大会` のような書き方）。
- **2日以上ある大会**（例：4月4日と5日の2日間）は、次のどの書き方でもOKです。
  書いたとおりにページへ表示され、並び順は**初日**が基準になります。
  ```
  2026年4月4-5日  ／  2026年4月4・5日  ／  2026年4月4〜5日
  2026年4月4日〜5日  ／  2026年4月30日〜5月1日（月をまたいでもOK）
  ```
- 日にちが決まっていない・書かない場合（例：`2026年4月`）は、**その月の1日**の扱いで並びます。
- 各シートの1行目は見出し：`種目 / 氏名 / 学年 / 記録 / 順位 / PB/SB`。
- 「順位」が1位の行は自動で金色（🥇）に、`PB` / `SB` は金色のバッジになります。
- タブ名の先頭を「_」にすると、そのシートは表示されません（メモ用など）。

### 大会の表示順について
- ページでは **開催日の新しい大会が自動で一番上** に表示されます。
- タブをどこに足しても順番は自動で整うので、いつも通り右端に足すだけでOKです。
- タブ名に日付が無い大会は末尾に表示されます。

### 年度を追加・設定する手順（メモ帳だけでOK）
1. その年度の大会結果スプレッドシートを用意する。
2. `apps-script/results.gs` の手順に従って、そのスプレッドシートにGASをデプロイし、
   **「ウェブアプリのURL」** を取得する。
   （`SPREADSHEET_ID` は **空欄のままでOK**。詳しくは results.gs 内の説明を参照）
3. `content/results.txt` を開き、1行足す：
   ```
   2027：https://script.google.com/……/exec
   ```
   書く順番は自由です（ページでは自動的に新しい年度が先頭になります）。
4. URLがまだ無い年度は `2027：` のように右側を空にしておけばOK。
   「未設定」の案内が出るだけでエラーにはなりません。

## 🔧 自己ベスト記録の「読み込みエラー」を直す（改善済み）

以前、自己ベスト記録ページで **「CORSエラー」** が出ることがありました。
これは実際にはサーバー側（GAS）で **エラーが起きて500が返っていた** ことが原因です
（500応答にはCORS許可が付かないため、ブラウザ上ではCORSエラーとして表示されます）。

`apps-script/personal.gs` を **「絶対に500を返さない堅牢版」** に差し替えました。
シートが空・列ズレ・IDの指定漏れなどがあっても、エラー内容をJSONで返すため、
ページは真っ白にならず案内メッセージが出ます。

**差し替え手順は `apps-script/personal.gs` の冒頭コメントに詳しく書いてあります。**
（要点：スプレッドシートを開く → Apps Script にコードを貼る → `SPREADSHEET_ID` を貼る →
「新しいバージョン」でデプロイし直す → アクセスは「全員」）

> ウェブアプリのURLが変わらなければ、`js/main.js` の `PERSONAL_URL` は変更不要です。
> もしURLが変わった場合のみ、`main.js` の `PERSONAL_URL` を新しいURLに直してください。

## 🧩 ヘッダー・フッターを直したいとき（共通・1ファイル管理）

ヘッダー（上のメニュー）とフッター（下のサイトマップ・SNS）は、
**`js/layout.js` の1ファイルだけ**で全ページ分を管理しています。
各ページの中には目印の2行しか無く、表示されるときに `layout.js` の内容が自動で入ります。

- メニュー項目やロゴ、フッターのリンクを変えたい → **`js/layout.js` だけを編集**すればOK。
  もう「8ページ全部を直す」必要はありません。
- ロゴ画像を差し替えるときも、`js/layout.js` 内の `logo.png` の1か所を直すだけです。

> ⚠️ 触るのは `js/layout.js` の上半分（ヘッダー／フッターの文章部分）だけにしてください。
> 一番下の「挿入処理」は変更不要です。

**そのほか今回の改善（自動で効きます・操作不要）**
- フッターのサイトマップは、**文字の上だけ**が押せる範囲になっています。

**ヘッダーの見た目（2026-07 改修後の仕様と調整場所）**

上のヘッダーは「白い角丸枠のロゴ（左端）」と「紫ピルのメニュー（右端）」の
2つの独立した部品でできていて、**帯そのものは透明**です。
ロゴとメニューの間の何もない部分はクリックが下に通るので、
スクロールしてヘッダーの下に入ってきたニュースのリンク等もそのまま押せます。

よく変えたくなる場所（すべて `css/style.css`。各行に【調整】コメントあり）：

| 変えたいこと | 場所 |
|---|---|
| 画面の端からの余白（今は18px） | `:root` の `--header-edge-gap` |
| メニューの紫の濃さ | `:root` の `--nav-pill-bg` / `--nav-pill-bg-hover` |
| ロゴの白枠の角丸・余白 | `:root` の `--logo-frame-radius` / `--logo-frame-pad` |
| ロゴの大きさ | `.site-logo img { height: … }` |
| スマホ全画面メニューの色・透け具合 | `.mobile-nav` の `background`（今は rgba(69, 20, 160, 0.97)） |
| ヒーロー写真の顔の位置 | `.hero-bg` の `object-position`（PC／スマホ／大画面で別指定） |
| 「上に戻る」ボタンが出るタイミング | `js/main.js` の `TOTOP_SHOW_AT`（今は300。0で常に表示） |
| 「上に戻る」ボタンの位置・大きさ | `css/style.css` の `.to-top`（`--totop-edge-gap` と width/height） |
| 起動アニメ（スプラッシュ）の長さ | `js/layout.js` の `SPLASH_TOTAL_MS`（今は1400ミリ秒） |
| 起動アニメの背景色 | `css/style.css` の `#app-splash` の `background` |

- カレンダー・地図・お問い合わせフォーム・記録の各ページは、読み込みが終わるまで
  **「表示には少し時間がかかる場合がございます。」** と表示し、表示され次第そのメッセージは消えます。

## 🚀 Netlify への公開・更新

1. 変更したファイルを保存する。
2. Netlify の「Deploys」タブ →「Deploy manually」→ サイトのフォルダごと再ドロップ。
3. 数秒で反映される。

> 📌 更新時は **フォルダまるごと** をドロップするのが確実です
> （`content/` や `images/` の更新漏れを防げます）。

> ⚠️ **公開してはいけないもの：`.git` フォルダ ほか**
> パソコン上のサイトフォルダに `.git` という隠しフォルダがある場合
> （GitHubなどで履歴管理しているとき）、**そのままドロップしないでください**。
> 過去の編集履歴（削除したはずの内容を含む）が丸ごと公開されてしまいます。
> **`.gitignore` や `.DS_Store` も、サイトの表示には使われないので上げません**
> （危険度は `.git` ほどではありませんが、上げる理由がありません）。
> - おすすめの手順：公開用に **これらを含まないコピーのフォルダ** を作り、
>   毎回それをドロップする（Macは Finder で隠しファイル表示：⌘+Shift+. ／
>   Windows はエクスプローラーの「表示」→「隠しファイル」で確認できます）。
> - 万一含めてしまっても、`netlify.toml` に外部から `.git` / `.gitignore` を
>   見えなくする保険のブロック設定を入れてありますが、**含めないのが基本**です。

> ❗ **`.gitignore` を書いてもドロップ公開では除外されません**
> `.gitignore` は **Git だけのルール**です。フォルダをブラウザにドロップする
> 公開方法では Git は動いていないので、何を書いても効きません。
> 除外は下の「公開用コピー」で行ってください。

### 公開用コピーの作り方（毎回これをドロップする）

手作業で消すより、下のコマンドを毎回1回叩くほうが確実です。
`-publish` の付いたフォルダができるので、**そちらをドロップ**します。

**Mac（ターミナル）**
```bash
rsync -av --delete \
  --exclude='.git/' \
  --exclude='.gitignore' \
  --exclude='.DS_Store' \
  --exclude='preview-*.html' \
  ~/ait-athletics/ ~/ait-athletics-publish/
```

**Windows（コマンドプロンプト）**
```
robocopy C:\ait-athletics C:\ait-athletics-publish /MIR /XD .git /XF .gitignore .DS_Store
```

> 💡 `--delete` / `/MIR` は「元で消したファイルはコピー先でも消す」という意味です。
> これがないと、整理して削除したはずの古い記事が公開先に残り続けます。

## 🔎 Google 検索に載せる（SEO・Search Console）

サイトを公開しただけでは、すぐには Google 検索に出てきません。
検索エンジンに「見つけてもらう」ための設定を入れてあります。以下は
**公開URLを GitHub Pages（`…github.io`）にする場合** の説明です。

### 正式な公開URLの決め方（GitHub Pages）
- GitHub Pages のURLは **「アカウント名.github.io」** の形で、`.github.io` の前は
  **GitHubのアカウント（ユーザーまたは組織）の名前**そのものです。
  リポジトリ名だけ変えてもURLは変わりません。
- 部活用には **組織（Organization）アカウント** を作るのがおすすめです
  （個人アカウントを残せる／卒業時の引き継ぎが権限の付け替えだけで済む）。
  1. GitHub右上「＋」→ **New organization**（Freeでよい）→ 名前を決める。
  2. その組織に、リポジトリ名 **「組織名.github.io」**（すべて小文字）で新規作成。
  3. サイト一式を `main` ブランチ直下に置く。
  4. リポジトリ Settings → **Pages** → Source を `main` /（root）に設定 →
     数分で `https://組織名.github.io/` で公開。
- ⚠️ このサイトは絶対パス（`/css/…`）なので、必ず **「アカウント名.github.io」の
  ユーザー／組織サイト（ドメイン直下）** にしてください。
  プロジェクトサイト（`/リポジトリ名/` の形）にすると表示が崩れます。
- ⚠️ URL（ホスト名）は **大文字小文字を区別しません**。CamelCase で登録しても、
  実際のサイトURLは小文字（例：`ait-athletics.github.io`）で動きます。

### 入れてあるファイル（さわらなくてOK）
- **`robots.txt`**（ルート）… 検索エンジンに「全部見てよい」と伝え、サイトマップの場所を案内。
- **`sitemap.xml`**（ルート）… サイト内の全ページの一覧。検索エンジンへの「地図」。
- 各ページの `<head>` の **`<link rel="canonical">`** … 「このページの正式URLはこれ」と
  伝える印。Netlify と GitHub Pages の**両方で同じ内容が見られても、GitHub Pages 側を
  正式扱い**にして重複と見なされないようにしています。

### 検索に載せる手順（公開後に一度だけ）
1. **Google Search Console**（無料）を開き、サイトの所有者確認をする
   （`…github.io` の場合は、指定されたHTMLファイルをサイトのルートに置く方式が簡単）。
2. 「サイトマップ」に **`sitemap.xml`** を送信する。
3. 「URL検査」で主要ページ（トップ・部員紹介・記録など）を入力し、
   **「インデックス登録をリクエスト」** を押す。
4. さらに **Bing Webmaster Tools**（無料）にも登録すると、Google 側の取り込みが
   遅いときの保険になります（Search Console の設定をそのまま取り込めます）。

> 💡 反映には時間がかかります（数日〜数週間）。GitHub Pages はサイトマップの
> 取得が遅いことがあるので、上の「URL検査で手動リクエスト」を併用してください。
> 確認は Google で **`site:組織名.github.io`** と検索して、自分のページが出れば
> インデックス済みです。

### ニュース記事を増やしたときのSEO作業
- 新しい記事を追加したら、その記事のURL（`…github.io/news/フォルダ名/`）を
  **`sitemap.xml` に1行足す**と、見つけてもらいやすくなります
  （面倒なら Claude に `sitemap.xml` を渡して「記事を追加したので更新して」と頼めばOK）。
- 記事ページ本体（`news-template.html` から作る）には canonical のひな形が入っています。
  Claude が記事を作るときに **フォルダ名に合わせて自動で設定** します。

## 🔐 サイトの安全について（大事なところだけ）

### 「サイトを勝手に書き換えられない？」への答え
- 公開後のサイト（.txt を含む）を **一般の訪問者が書き換えることはできません**。
  訪問者ができるのは「見る」ことだけです。
- 書き換えられるのは、**Netlify にアップロードし直せる人＝部の担当者だけ** です。
  つまり守るべきは「アカウント」です。次を必ず守ってください。
  - **Netlify・Google アカウントに2段階認証を設定する**（最重要）。
  - パスワードは使い回さない。編集・公開の権限は担当部員のみに絞る。
  - 卒業時は Netlify・スプレッドシート・フォーム・カレンダーの権限を後任へ引き継ぐ。

### 「公開される」ことを前提にした情報の考え方
- このサイトは公開サイトなので、**content/ に書いた内容・images/ の写真は
  すべて誰でも見られます**（クリックで開く「出身県・学科」も同じです）。
- **公開してよい情報だけを書く** のが唯一確実な守り方です。
  迷う情報（詳しい住所・電話番号・生年月日など）は書かないでください。
- 連絡先メールは前半・後半に分けて書く方式にしてあり、
  アドレス収集ボット対策になっています（上の site.txt の説明参照）。

### 文字コピー・画像保存の抑止（入れてあります）
- サイトの文字は **選択・コピーできない** 設定です
  （お問い合わせフォームの入力欄と、ニュース整理ページのコピー用ブロックは除く）。
- 画像は **右クリック保存・スマホの長押し保存・ドラッグ保存ができない** 設定です。
- ⚠️ ただしこれは「気軽な持ち出しの抑止」です。**スクリーンショットや
  ページのソース表示、画像URLへの直接アクセスまでは技術的に防げません**。
  本当に出したくない文章・写真は、そもそも載せないでください。

### `.gitignore` について（GitHub を使う場合・さわらなくてOK）

`.gitignore` は **「GitHub の管理に入れないファイル」のリスト**です。
サイトフォルダの一番上（`index.html` と同じ場所）にあります。

- **役割**：`.DS_Store`（Macが勝手に作る）やバックアップ、`publish/`（公開用コピー）
  など、サイト本体ではない物が GitHub に紛れ込むのを防ぎます。
- **Netlify のドロップ公開には効きません**（上の「公開用コピー」を参照）。
- **中身を推測させる名前を書かないこと**（例：`名簿.xlsx`、`部員連絡先` など）。
  `.gitignore` 自体は見られる可能性があるため、書くと「そこに何かある」と
  教えることになります。**見せたくないファイルは、そもそも
  サイトフォルダの外に置いてください**（これが唯一確実な守り方です）。
- **書き足したいとき**：1行に1つ、ファイル名やフォルダ名を書きます。
  フォルダは末尾に `/` を付けます（例：`メモ/`）。
  `#` で始まる行はコメントです。**`#` は行頭だけがコメント**で、
  `publish/  # 説明` のように後ろに書くと効かなくなるので注意してください。

> ⚠️ **すでに GitHub に入れてしまったファイルには効きません。**
> `.gitignore` は「これから追加する物」だけが対象です。
> 既に入っている物を外したいときは Claude に相談してください
> （`git rm --cached` という操作が別途必要です）。

### 編集ミスでサイトが壊れない仕組み（入れてあります）
- content/ の .txt が読み込めない・書式が崩れている場合でも、
  **そのセクションにだけ案内が出て、ページ全体は壊れません**。
- それでも、**書き換える前に元ファイルのコピーを取っておく** と安心です
  （例：`members.txt` をコピーして `members（バックアップ2026-07-07）.txt` を
  自分のパソコンに残しておく。※コピーを `content/` に入れたまま
  アップロードしても表示には影響しませんが、置き場所は分けるのがおすすめ）。

## ✨ 起動アニメーション（スプラッシュ）について

- サイトを **そのセッションで最初に開いたとき1回** と、**ページを再読み込み（リロード）したとき**、
  **ヘッダーのロゴを押したとき** に、
  紫の画面中央にロゴが浮かぶ短いアニメーション（約1.4秒）が流れます。
- **ページを移動するたびには出ません**（別ページへのリンク移動・戻る/進むでは出ない。閲覧のテンポを守るため）。
  リロード（F5・⌘+R・再読み込みボタン）でだけ再生されます。
- OSで「視差効果を減らす」を設定している人には表示されません（体調への配慮）。

## ❓ 困ったとき

- **文字が反映されない** → HTMLを直接ダブルクリックで開いていないか確認（冒頭の注意参照）。Netlifyで確認する。
- **部員の人数がおかしい** → `members.txt` の `---` の入れ忘れ・入れすぎを確認。
- **大会結果が「未設定」と出る** → `content/results.txt` にその年度のURLが入っているか確認。
- **記録が「読み込めませんでした」と出る** → GASのデプロイ（新しいバージョン／アクセス「全員」）を確認。
- **メールアドレスが「(読み込み中…)」のまま** → `content/site.txt` の
  「連絡先メール前半」「連絡先メール後半」が両方書かれているか確認。
- **スマホでメニューの下に白い隙間が出る／ボタンを2回押さないと反応しない** →
  改善済みです。まだ起きる場合は、古いキャッシュの可能性があるので
  ページを再読み込み（またはブラウザのキャッシュ削除）してください。
- それでも解決しないときは、状況をそのまま Claude に伝えて相談してください。
